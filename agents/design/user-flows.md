# User Flows

Status: DONE
Owner: UI/UX Pro Agent
Tool mode: mcp_preferred
MCP usage: not used in this baseline pass; flows derived from current repository structure and planned acquisition model.

## Objective

Define the minimum viable user journeys from discovery to enquiry so design, content, and development work can support conversion without ambiguity.

## Flow 1: Homepage to contact enquiry

1. User lands on homepage from a broad commercial query.
2. User confirms that the company handles renovation from engineering to finishing.
3. User scans service breadth, process, and trust sections.
4. User selects primary CTA.
5. User reaches contact page.
6. User sees the expected project brief fields.
7. User decides to make contact with enough clarity to describe the project.

### UX requirements

- headline must explain full-scope offer immediately
- CTA must appear above the fold
- trust blocks must support the CTA rather than distract from it

## Flow 2: Service-specific search to enquiry

1. User lands on a service detail page such as plumbing or electrical.
2. User confirms the exact service is covered.
3. User reviews scope, inclusions, and where that service fits into a wider renovation.
4. User sees how the team coordinates adjacent stages.
5. User moves to contact or related service page.

### UX requirements

- service page must connect the specific trade to the full renovation system
- CTA should reference the current service context
- related links should help users continue exploring without getting lost

## Flow 3: Trust-building path for cautious buyer

1. User lands on homepage or about page.
2. User wants proof of methodology before contacting.
3. User visits process page, about page, and FAQ.
4. User understands how scope, control, and supervision work.
5. User proceeds to contact page with reduced uncertainty.

### UX requirements

- process and FAQ must be accessible from primary navigation
- content should reduce anxiety, not create marketing noise
- the site must feel operationally competent

## Flow 4: Mobile-first fast qualification

1. User opens the site on mobile.
2. User scans the header, hero, and first one or two sections.
3. User decides whether the company is relevant.
4. User taps either Services or Contact.

### UX requirements

- headline, service summary, and CTA must remain visible quickly on mobile
- navigation must stay simple
- sections must not become long walls of text

## Conversion blockers to avoid

- vague value proposition
- buried CTA
- fragmented service presentation
- lack of process clarity
- contact page that does not guide the brief

## Notes for next phases

- Gate 2 should convert these flows into component-level and section-order decisions.
- Development should preserve short paths from homepage, service pages, and trust pages into contact.
